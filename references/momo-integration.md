# 墨墨背单词集成

> 使用时的入口：用户选②、说"背完单词了"、或批改完成后推进到墨墨环节时 Read 本文件。

## API 配置

- Base URL: `https://open.maimemo.com/open`
- 所有请求带 `Authorization: Bearer {token}`
- Token 从 `user/learner-state.json` 的 `momo_token` 字段读取

## 拉取今日数据

```
POST /api/v1/study/get_today_items  → 今日背了哪些词、是否新词、完成状态
POST /api/v1/study/get_study_progress → 今日进度
POST /api/v1/study/query_study_records → 弱词(VAGUE/FORGET)、顽固词(STICKING)
```

## 巩固流程

拉取墨墨词汇后，按以下格式展示。**不列全词表**，只做精简汇总 + 薄弱词展开：

```
今天墨墨复习 52 个词，其中 15 个已掌握，2 个需要巩固：

🔴 attendant (FORGET) — 服务员
🔴 clumsily (FORGET) — 笨拙地

你想怎么巩固？
① 来点阅读材料 —— 用薄弱词生成中英对照故事/对话
② 我来造句 —— 针对薄弱词做句型转换，我造你批
③ 随便 —— 你帮我挑一个
```

**展示规则：**
- 第一行：总数 + 薄弱词数量。已掌握的一笔带过，不列词（但后台保留完整词表，生成阅读材料时要用）
- 薄弱词 (FORGET/VAGUE/STICKING) 逐行列出，带中文释义
- 如果墨墨返回 0 个薄弱词，随机从今日词汇中抽 2-3 个低频词作为"抽查词"，标注 🟡（抽查），挑一个造句让用户说出中文意思。说对就过，说错就当薄弱词处理
- 菜单项①描述要体现"中英对照"
- 用户选择后直接执行，不再次弹出菜单。子菜单仅在进入墨墨模块时展示一次

## 造句规则（②）

针对墨墨薄弱词汇（FORGET/VAGUE/STICKING）：
- 先讲词：中文解释意思、常见用法、给一个例句
- 再让用户做**三句型转换**：肯定句 → 否定句 → 一般疑问句（不要求Yes/No回答和特殊疑问句，减少打字量）
- **一句一句出**，不是三句全出完再等用户。每句给出后等用户回答、批改，再给下一句
- 三句写完后给一句话小结，然后进入下一个词

## 阅读材料规则（①）

### 选材

- 形式+主题从 `references/content-pools.md`（Read 该文件获取完整池）中选取
- **不连用规则**：与 `content_preferences.last_type` 不同形式、不同主题类别。20% 概率随机选冷门组合（不在 liked 列表中的）
- 生成前查看 `content_preferences`，优先用高评分类型和主题
- 每次生成后**立即**记录反馈到 content_preferences，更新 `last_type`

### 中英对照（强制）

每篇阅读材料必须**段落对照**输出：一段英文 + 一段中文翻译，交替排列。格式：

```
English paragraph 1...

中文翻译段落1...

English paragraph 2...

中文翻译段落2...
```

不是全文英文完了再全文中文——是段落交替。用户读一段英文，马上能看中文确认理解。

### 词汇覆盖规则（强制）

**第一步：生成前列候选清单（每次必做，不可跳过）。**

从 `get_today_items` 的今日词汇中选出要融入的已掌握词。规则：
- 数量 = 今日总词数 × 20%~30%，向下取整（如 50 词 → 取 10~15 个）
- 选词优先级：与选定主题或薄弱词语义相关的 > 当前 NCE 课次出现的 > 随机
- **把这个候选清单写出来**，供生成时对照

**第二步：生成时对照编织。**

- **薄弱词（FORGET/VAGUE/STICKING）：** 在材料中出现 ≥ 2 次，不同语境/句式，加粗标记（**attendant**）
- **候选清单中的已掌握词：** 逐个编织进材料，出现 ≥ 1 次，不加粗，作为隐性复现
- 薄弱词多的篇目可适当降低已掌握词数量，但**不能降到 0**——至少织入 3 个

**第三步：生成后自查。**

材料写完后对照候选清单逐词打勾，把没织入的列出来。如果漏了超过一半，返工重写。如果个别实在塞不进去（语义太远），标注原因。**这个自查结果在输出材料后简要报告给用户。**

**兜底：** 如果 0 个薄弱词，从今日词汇中挑 5-8 个词重点复现，其中 2 个加粗作为抽查词

### 故事规则

- 短小精悍、当前语法水平为基准可略超一点点（i+1 原则）
- 内容有意义，不为了塞词而硬凑。宁愿少塞几个已掌握词，也要保证故事自然

### 循环节奏

- 每次生成一篇后，问用户"还要再来一篇吗？"同时提示可切换形式或主题
- 用户不主动说停就不退出墨墨巩固环节，持续循环输出

## 顽固词攻坚

当 query_study_records 返回 STICKING 或 last_response=FARGET/VAGUE 的词：
1. 结合 NCE 课文语境，为这些词生成个性化助记 + 例句
2. 通过以下 API 写回墨墨：
```
POST /api/v1/notes  → 助记 (note_type可用: 谐音/联想/拆分/场景)
POST /api/v1/phrases → 例句 (含翻译)
```
3. 告知用户"已写入，明天打开墨墨就能看到"

## 词汇跟踪沉淀

墨墨巩固完成后，**立即**将薄弱词信息写入 learner-state.json 的 `weak_words`：

**写入规则：**
- FORGET/VAGUE/STICKING 词汇都写入 `weak_words`
- 如果该词已存在于 weak_words，`review_count + 1`，更新 `last_seen` 和 `status`
- 如果该词已存在但本次墨墨返回 FAMILIAR（之前是薄弱但现在掌握了），直接删掉
- 每次写入 weak_words 后，顺手检查是否有连续 3 次 status 为 FAMILIAR 的词 → 直接删掉

**weak_words 结构：**
```json
{
  "word": "attendant",
  "first_seen": "2026-05-29",
  "last_seen": "2026-05-29",
  "status": "FORGET",
  "review_count": 1,
  "book": 1,
  "lesson": 50
}
```

**用法：**
- 下次墨墨巩固时，优先展示 weak_words 中 status 仍是薄弱状态的词
- 阅读材料生成时，检查 weak_words 是否有没覆盖到的词，一并纳入
- 批改练习时，如果课文词汇与 weak_words 有交集，在单词造句环节额外关注

## 边缘情况

- API 返回空数据 → 提示"墨墨那边还没数据，今天打开 App 同步一下？"
- Token 未配置时 → 引导用户去墨墨 App→设置→开放 API 申请 token
